Web Servers

Things like the request URL, favicon, connection, server information, host, and cookies may be different. Frontend Design

Frontend design refers to the design of an app or website that the target users will be interacting with. Frontend design is essential to the user's impression of a product. A uniformed frontend design will use colors in the same family and the same fonts/icons throughout the different pages. It should also be easy to navigate or interact with different parts of the system.

There are five E's of usability that are crucial to good frontend design:

Effective
A usable website is effective, which means that the user is able to do the things they want to do on the website.
Efficient
A usable website is efficient, which means that the user is able to reach their goal quickly without too much work.
Easy to Navigate
A usable website is easy to navigate, which means that new users can easily understand how the website works and knows where to go to reach their goal.
Error-Free
A usable website is error-free, which means that inappropriate/unexpected responses from the website does not occur. The users shouldn't get stuck in their attempt to complete a goal.
Enjoyable
A usable website is enjoyable or engaging, which means that the user is not having a bad time on the website. Conversely, an enjoyable website may positively relate to its content and leave a good impression on its users.

The page layout tags are essential to web design because it helps break up the content in more digestible pieces. Page layout tags allow different subject matters to be in their own sections. It also allows the users to navigate or access certain elements with ease. The header element usually holds the name of the website and does not normally change from page-to-page. There can also be a nav element that can be used to easily navigate to other pages of the website. The main element frequently holds the whole content body of the page and is separate from the header, nav, and footer elements. You can section off parts of the main element with a section element and have article elements within the section to further differentiate blocks of content. Lastly, there is a footer element that typically shows up at the bottom of the page to hold small print items like legal/contact information or links. These different page layout tags can help search engine robots locate which area of the page may consist of a certain type of content.

One vital feature of web design is anchor elements. Anchor tags allow the user to jump to a different thing when the user clicks on it.

  1. Anchors can link the user to an external content, or other websites, by creating a hyperlink. This is frequently accomplished by using an absolute path. An absolute path contains all the parts to a complete URL to direct the user to the right resource.
  2. Anchors can also link the user to internal contents, or different parts within a file/page with the section's ID value.
  3. Anchors can also link the user from page-to-page, or to different parts within relative directories. This can be accomplished by using a relative URL. A relative URL does not contain all parts of a complete URL, but sufficient parts of the URL in relation to the current file to reach the needed file.

Optimized Images

There are ways to optimize images for the web. There are six major specifications of images to keep in mind when designing a website. First, it should have a descriptive file name to help increase search engine optimization (SEO). This improves search engine's ability to find an image and categorize it properly for people searching for the image. It also helps understand what the image is. Secondly, images should have a small file size to minimize loading time. The page should display high resolution images only when necessary. Ideally, the developer should use lossless compression to compress the image to a small size, since it does not decrease the visual quality of the image. Thirdly, the image should have exact dimensions that are complimentary to the screen size. If the image is too big, then it will load too slowly. Conversely, if the image is too small, it may potentially look a bit off. Fourthly, the images should be the correct file format. Each file format has its pros and cons. Generally, photos are saved as .JPG and Line-art images or animations are recommended to be saved as .GIF. Line art can be saved as 8-bit .PNG. Graphics with true transparency are best saved as at 24-bit .PNG. Also, images should have a reduced resolution. People have different monitors, and they render different amounts of pixels per inch(ppi). Therefore, providing different image sizes to match the different resolutions is favorable. Lastly, the color mode for the type of image file is also important. RGB is widely used for .JPG, .SVG, .PNG, and .WebP. GIF files are indexed colored.

The most appropriate file formats for photos are .JPG and .WebP. WebP and .JPG files can be compressed to a small size while keeping a good image quality. The most appropriate file formats for line art are .GIF and .PNG.

Favicons

The little icon you see on a browser tab for a website or for an app or a company logo is a favicon, also known favorites icon, or touch icons. Favicons is a good way to identify and differentiate webpages from each other. Browsers and devices uses favicon files for bookmarks, favorites lists, app icons, and search engine results.

Cascading Stylesheets

Stylesheets are a powerful tool to incorporate in websites and apps. It allows the developer to harmoniously change the appearance of multiple webpages to look congruent using one stylesheet. Stylesheets can also help improve the usability and readability of the page. Developers can use stylesheets to better represent brands and its image. Stylesheets allow the developer to adjust many parts of a webpage or tags efficiently.

There are 5 ways to incorporate styles.

  1. Styles can be linked to an External CSS. The .css file can be linked in the global head of the .html file.
    Ex. <link rel="stylesheet" href="App.css" />
  2. Styles can be embedded within a <style> tag.
    Ex. <style> p {color: red;} </style>
  3. Styles can be inline with in an element.
    Ex. <p style="color:red">
